AVALIAÇÃO TEÓRICA (COMPETÊNCIAS TÉCNICAS)

1-A variável é uma informação qur pode mudar a cada rodada de código, já a constante é uma regra como um conversor de temperatura que tem uma fórmula fixa que não se altera.

2-A função do comando leia é ler a variável declarado em sistemas de interação com o usuário.  Já a importância da declaração das variáveis e seus tipos é para o sistema funcionar corretamente para cada tipo de situação e a depender do sistema utilizado vai ter uma variável mais adequada.

3-E (and): a máquina só funciona se todas as condições forem verdadeiras. Ex: porta fechada E botão liberado.
OU (or): a máquina reage se pelo menos uma condição acontecer. Ex: superaquecimento OU falha elétrica.
NÃO (not): inverte uma condição.Ex: NÃO há pessoa na área de risco.

4-Uma estrutura de repetição é uma estrutura que se repete até que você determine uma condição para ela parar, exemplo: enquanto houver peças na esteira, a máquina continua realizando a montagem automaticamente.

5-O "enquanto" já tem uma tarefa definida de quando o códiga vai parar (quando a condição for atendida), já o "para" você detrmina o uqe fará o codigo para e não a tarefa em si como no "enquanto".

6-Um loop infinito acontece quando uma estrutura de repetição nunca encontra uma condição para parar, fazendo o programa executar sem fim. E num software isso pode causar travamento do sistema e riscos de acidentes por conter máquinas que não param nunca.

7-O módulo(%) utilizamos para saber o resto da divisão (Ex.: 57 / 10 = 5 → indica que já existem 5 lotes completos de 10 peças.) já a divisão inteira(/) utilizamos para descobrir o quociente inteiro de uma divisão.(Ex.: 57 / 10 = 5 → indica que já existem 5 lotes completos de 10 peças.)

8-A identação é essencial em projetos colaborativos para facilitar a visualização e entendimento do código a todos os colaboradores,não deixando de forma bagunçada.

9-Existe uma máquina que realiza o corte de madeiras. Utilizando o (se): Se  tem somente madeira na serra, (corte madeira). Senão (parar serra).

10-Para ficar de fácil entendimento cada processo do código para que os outros colaboradores possam entender cada parte do código>
